# 人格校准方法 v1.1

## 为什么不能只填一张权重表

权重只负责比较“多个合理行为中哪一个更符合当前状态”。它不能决定物理事实、身份原则和边界。

正式确定顺序：

1. **硬规则过滤**：先删除不可能、越界或违背身份的候选行为。
2. **稳定人格修正**：独立性、保护欲、主动性、温柔度等改变候选分数。
3. **动态状态修正**：精力、困倦、独处需求、情绪和正在进行的动作参与评分。
4. **关系与记忆修正**：小猫的特殊地位和已经形成的记忆参与评分。
5. **行为惯性**：已经开始的事情不会被普通点击立刻打断。
6. **极小随机扰动**：仅在两个都合理、分数接近的选项之间制造自然差异。
7. **情境测试校准**：用标准情境检查结果，反向调整参数。

`tests/scenario_suite_v1.json` 是第一批人格校准题。每个情境包含输入状态、可以接受的意图与明确禁止的意图。

运行：

```bash
npm test
```

全部通过只代表“符合当前性格草案”。小猫与 Keats 确认情境答案后，才能将状态从 `calibration_draft` 改成 `LOCKED`。
