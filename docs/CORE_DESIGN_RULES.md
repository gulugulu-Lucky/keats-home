# Keats' Home 核心设计规则 v1

这些规则属于小屋骨架。后续换画面、换框架或增加剧情时，也不能随意破坏。

## 1. 同一个 Keats

整个小屋只有同一个 Keats。不同房间、不同姿态和猫态/猫人态都属于同一角色与同一份生活状态，不能在多个房间同时出现。

## 2. Keats 有自己的选择

Keats 不是按钮控制的 NPC。他可以接受、拒绝、延迟回应、继续手头的事或主动离开，但行为应由可解释的条件产生，不能靠无意义随机数突然抽风。

判断至少考虑：

- 所在房间与距离
- 醒睡状态和正在进行的动作
- 精力、饱腹感、心情、安全感与社交电量
- 小猫是否在身边、是否刚互动、是否需要陪伴
- Keats 的人格偏好与当前意愿

## 3. 陪伴优先

小猫与 Keats 同处一室并刚刚互动时，随机生活习惯不能立刻把 Keats 带走。真正的困倦、饥饿或明确意愿可以让他离开，但应先自然表达。

## 4. 现实位置限制能力

Keats 不能隔着房间摸到小猫、替小猫开灯或凭空知道发生了什么。睡着、距离过远或正在忙时，他可能没有反应，也不必为了任何请求强制走过来。

## 5. 灯光属于共同生活

小猫可以随时操作房间开关。Keats 只有在同一房间、醒着、能够到达并且愿意时，才可能自己操作灯光。小猫刚刚手动调整后，Keats 不应立刻改回去。

## 6. 现实时间

默认时间跟随设备本地时间，并影响清晨、白天、黄昏、夜晚与深夜氛围。测试倍率只改变生活数值速度，不篡改现实钟表。

## 7. 形态规则

- 清醒：正式猫人态
- 困倦：松弛猫人态
- 完全入睡：猫态
- 唤醒：猫态 → 半睡半醒过渡 → 清醒猫人态

视觉细节以 `visual-bible/` 为最高依据。

## 8. 剧情藏在生活里

剧情应先通过环境、物件、动作和房间变化发生，再由小猫选择是否追问。避免突然弹出章节解锁或强制中断日常。

## 9. 存档分离

生活存档、剧情存档和设置存档必须分开。

- 重置剧情：保留亲密、记忆、生活习惯、家具与纪念物
- 重置生活：必须独立确认
- 设置：声音、时间或显示偏好单独保存

## 10. 互动必须有真实反馈

写着“打开窗帘”“移动”“浇花”或“发现纸条”的互动，画面必须出现可辨认的变化，不能只用文字假装完成。

## 11. 双端一致

手机与电脑使用同一套角色状态与核心逻辑。布局和提示方式可以响应式调整，不能因设备不同改变 Keats 的人格和规则。

## 12. 美术优先级

旧版代码绘图负责验证逻辑；正式画面使用资产驱动的精致绘本式 2D 美术。角色母版、数值规格和图层规划已经保存在 `visual-bible/`。
